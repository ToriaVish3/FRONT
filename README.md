# GlassCloud Frontend (Frontend-only)

## Что изменено
Проект очищен от бэкенд-логики (Supabase, авторизация, загрузка файлов, таблицы и миграции). Остался только фронтенд-интерфейс и навигация между страницами.

Удалено:
- `src/services/*` (логика авторизации и обращение к БД)
- `src/lib/supabase.ts`, `src/lib/database.types.ts` (клиент Supabase и типы БД)
- `src/components/FileManager.tsx` (файловый менеджер с БД/Storage)
- `supabase/` (миграции)
- `.env` (ключи доступа)

## Карта файлов и логики (подробно)

### `src/pages/Home.tsx`
Главная страница с модальными окнами входа и регистрации.
- `showLoginModal`, `showRegisterModal`: управляют открытием/закрытием модалок.
- `loginNickname`, `loginPassword`: локальные значения полей «Ник» и «Пароль» в форме входа.
- `registerNickname`, `registerPassword`: локальные значения полей в форме регистрации.
- `error`: текстовое сообщение об ошибке/успехе, отображается в модалке.
- `handleLogin(...)`: 
  - раньше: отправлял ник/пароль на сервер через `loginUser` (см. удалённый `src/services/auth.ts`).
  - сейчас: только проверка заполненности и переход на `/home/cloud/:nickname`.
- `handleRegister(...)`:
  - раньше: отправлял ник/пароль на сервер через `registerUser` (см. удалённый `src/services/auth.ts`).
  - сейчас: только проверка заполненности и вывод «Регистрация успешна!».
- Блоки разметки:
  - Хедер с логотипом и кнопками (стеклянный эффект задаётся в CSS).
  - Модалки входа/регистрации, затемнение только вне формы.

### `src/pages/Cloud.tsx`
Страница «облака», теперь фронтенд-версия без БД.
- Получает `username` из URL (`/home/cloud/:username`).
- Показывает логотип GlassCloud, поле с ником по центру и кнопку «Выйти».
- `handleLogout()` — просто возвращает на `/`.

### Удалённая логика файлов/БД (что где было)
Раньше управление файлами/папками делалось в `src/components/FileManager.tsx` (удалён).
Там были функции:
- `handleAddFolder()` — создание папки в БД.
- `handleFileSelect()` — загрузка файла в storage и запись в БД.
- `handleDelete()` — удаление файла/папки из БД и storage.
- `loadItems()` — загрузка списка файлов/папок из БД.
- `handleDownload()` / `handleShare()` — скачивание и шаринг.

Связанные файлы (тоже удалены):
- `src/services/*` — авторизация и запросы к БД.
- `src/lib/supabase.ts` — клиент Supabase.
- `src/lib/database.types.ts` — типы БД.

### `src/components/AuroraRibbonHero.tsx`
Фоновый эффект (ленты/градиенты). Полностью визуальный слой, логики нет.

### `src/components/ui/*`
UI-компоненты (button, card, input, label). Используются на Home (и могут использоваться в других местах).

### `tailwind.css`
Глобальные стили:
- стеклянный эффект для кнопок и поля никнейма (glassmorphism)
- анимации и базовые стили Tailwind

### `src/index.tsx`
Входная точка React и роутинг:
- `/` → `Home`
- `/home/cloud/:username` → `Cloud`

### `index.html`
Корневой HTML. Только контейнер `#app` и подключение `src/index.tsx`.

## Про папку `src/screens`
Сейчас **не используется** в роутинге или импортах.
В `src/index.tsx` есть только маршруты на `Home` и `Cloud`, поэтому `src/screens` — лишняя папка и её можно удалить, если вы её не используете.

## Если понадобится вернуть бэкенд
Нужно будет вернуть:
- `src/services/*`
- `src/lib/supabase.ts`
- `.env` с ключами
- компоненты с логикой БД/Storage
